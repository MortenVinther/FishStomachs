Database_format

Table 1. database format. “field” is variable name used in FishStomachs; “types” is the type of input(c=character, i=intger, d=double, l=logical); “mandatory” is the field required as minimum (T=TRUE, F=FALSE); “PRED” is the variable part of predator information; “PREY” is the informtion part of prey information; “units” is the unit or values applied.

field types mandatory PRED PREY units remarks
dataset c T T F free text Data set name
country c T T F ICES Alpha code Sampling country
ship c T T F ICES Alpha code Sampling platform, e.g. Research vessel name
lat d F T F Sample lattitude, decimal format
lon d F T F Sample lattitude, decimal format
est_lat_lon c F T F Is longitude or lattitude estimated?
rectangle c T T F ICES statistical rectangle, or other smaller area
area c T T F ICES area, or other larger area
year i T T F yyyy Sampling year
quarter i T T F [1-4] Quarter of the year
month i T T F [1-12] Month
day i T T F [1-31] Day of the month
time i F T F HHMM Time of the day (24 hours)
station c T T F free text Sampling station
haul c T T F free text Sampling haul
samp_meth c F T F free text Sampling method, e.g. DEM=Demersal, PEL=Pelagic
depth d F T F meter Ambient depth
temp d F T F degree Celcius Ambient temperature
sample_id c T T T free text Unique id for one or more individual stomachs or pool of stomachs from the same sample station (e.g. haul ). Unique within the full dataset, e.g. created from combination of pred_name, pred_l, year, month, day, country, ship, haul, station, fish nr
fish_id c T T T free text Id for each individual stomach or pool of stomachs. Unique id in combination with sample_id
record_type c F T F SS|PS Collation of stomachs by individual stomach or by a pool of stomachs, SS=Single stomachs, PS=Pooled stomachs
pred_aphia i F T F WoRMS aphia id World Register of Marine Species code
pred_nodc d F T F NODC 10 digits NODC coding system, obselete but usefull
pred_name c T T F text Predator taxon, latin name
pred_w d F T F gram Body weight of predator
pred_age i F T F years Age of predator
pred_l i T T F mm Length of predator
pred_ll i T T F mm Predator length lower bound
pred_lu i T T F mm Predator length upper bound
pred_cpue d F T F Catch Per Unit effort, or other weighting factor for collation of data
gall_bladder c F T F Gall bladder stage
preservation c F T F Method of stomach preservation
n_tot i F T F Total number of stomachs in the pool. For single stomach always 1
n_food i T T F Number of stomachs with food (and not regurgitated)
n_regur i T T F Number of regurgitated stomachs
n_skel i T T F Number of stomachs with skeletal remains only
n_empty i T T F Number of empty stomachs
stom_content_w d F T F gram Total weight of stomach contents
stom_empty_w d F T F gram Weigth of stomach with no food left
stom_fullness d F T F % Stomach fullness ?????
prey_item c F F T Prey name as used in the original dataset
prey_aphia i F F T WoRMS aphia id World Register of Marine Species code
prey_nodc d F F T NODC 10 digits NODC coding system, obselete but usefull
prey_name c T F T Prey taxon, e.g. latin name
prey_group c F F T Name used for grouping of “prey_name”, e.g. broader taxonomic group
prey_ident_met c F F T ????
digest i T F T 0|1|2|3|9 Prey digestion stage 0= Intact prey (skin, fins, legs and flesh is complete), 1= partially digested prey (prey in more advanced stages of digestion), 2= partially digested prey (prey in more advanced stages of digestion), 3= skeletal material (no flesh, only bones, shells, otoliths)
prey_l_ident c F F T Prey length identifier ?????
prey_l d T F T mm Length of an individual prey
prey_ll i T F T mm Prey length lower bound
prey_lu i T F T mm Prey length upper bound
prey_w_meth c T F T r|p Method of weiging and recording the weights of preys items. r=by record, p=by prey species and digestion stage
prey_n i T F T Number of preys (whole animals) , or number of preys counted for weight_method equal to “p”
prey_w d T F T gram Weight of prey (see prey_weight_method)
prey_min_n c F F T Minimum number of preys or flag ?
remarks c F T F Free text
